# Objetivo do Dia

Compreender o problema que o sistema pretende resolver antes de iniciar qualquer desenvolvimento.

Nesta etapa foram levantadas as primeiras ideias sobre o produto, seu público-alvo, funcionalidades e escopo inicial.

---

# O Problema

Empresas normalmente possuem informações distribuídas entre diferentes fontes, como planilhas Excel, bancos de dados e APIs.

Essa descentralização dificulta:

- localizar informações importantes;
- gerar indicadores rapidamente;
- tomar decisões baseadas em dados;
- responder perguntas sobre o negócio de forma simples.

O objetivo da plataforma é centralizar esses dados, organizá-los e disponibilizar consultas inteligentes utilizando Inteligência Artificial.

---

# Público-Alvo (Personas Iniciais)

Inicialmente foram identificados os seguintes perfis de usuários:

- Analista de Dados
- Gestor
- Diretor
- Administrador do Sistema

Essas personas poderão ser refinadas durante o levantamento completo de requisitos.

---

# Funcionalidades Levantadas

As primeiras funcionalidades identificadas foram:

- Importação de arquivos Excel;
- Importação de arquivos CSV;
- Importação de dados por APIs;
- Conexão com bancos de dados;
- Organização dos dados importados;
- Geração de dashboards;
- Geração de relatórios;
- Consultas utilizando IA.

Estas funcionalidades representam apenas uma visão inicial e poderão sofrer alterações durante o projeto.

---

# Fora do Escopo (Versão MVP)

Nesta primeira versão o sistema **não terá como objetivo**:

- Controle de estoque;
- Sistema financeiro;
- Emissão de notas fiscais;
- Controle de folha de pagamento;
- Funcionalidades de ERP.

O foco será exclusivamente na análise, organização e consulta de dados.

---

# Lições Aprendidas

Durante a primeira reunião de levantamento de requisitos foram identificados alguns pontos importantes:

- O desenvolvimento de software começa entendendo o problema do negócio.
- Antes de escolher tecnologias é necessário compreender as necessidades do usuário.
- Definir o que **não faz parte** do projeto ajuda a controlar o escopo.
- Pensar no usuário é tão importante quanto pensar na implementação técnica.
